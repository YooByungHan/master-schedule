# HTTPS 설정 안내 (폴더 선택 저장 활성화)

## 왜 하나요?

`메뉴 → 출력 → EXCEL VB` 로 엑셀을 내보낼 때, **접속한 사용자 본인 PC에**
저장 위치를 **매번 직접 골라서** 저장하려면 서버가 **HTTPS**로 실행되어야 합니다.

- 브라우저의 "폴더 선택 저장" 기능은 보안 규칙상 **`https://` 또는 `localhost`**
  에서만 동작합니다.
- 지금처럼 `http://서버IP:3000` 으로 접속하면 이 기능이 막혀, 저장 위치를
  고를 수 없고 브라우저 기본 다운로드 폴더로만 내려갑니다.

서버를 HTTPS로 켜면 **모든 사용자에게 한 번에 적용**됩니다. (사용자별 개별 설정 불필요)

---

## 설정 방법 (서버 PC에서 딱 한 번)

서버를 돌리는 PC에서 명령창(PowerShell 또는 CMD)을 열고, 이 폴더로 이동한 뒤
아래 3줄을 차례로 실행하세요.

```bash
# 1) 필요한 모듈 설치 (인증서 생성용)
npm install

# 2) 자체 서명 인증서 생성 → certs/server.key, certs/server.crt 가 만들어집니다
npm run gen-cert

# 3) 서버 다시 시작
node server.js
```

서버가 뜰 때 이렇게 나오면 성공입니다:

```
 MASTER SCHEDULE 서버 v5 [HTTPS]
 ...
 🔒 HTTPS 모드 — 폴더 선택 저장 사용 가능
```

이제 사용자들은 `https://서버IP:3000` (앞에 **s** 붙음) 으로 접속합니다.

---

## 처음 접속 시 "안전하지 않음" 경고가 떠요 (정상입니다)

사내 전용 **자체 서명 인증서**라서, 브라우저가 처음 한 번 경고를 띄웁니다.
공개 웹사이트가 아니라 **우리 서버**에 접속하는 것이므로 안전합니다.

- **크롬/엣지**: `고급` → `(안전하지 않음) 계속 진행` 클릭
- 한 번 넘어가면 이후에는 정상적으로 접속됩니다.

> 경고를 아예 없애고 싶다면(선택): `certs/server.crt` 파일을 각 PC의
> "신뢰할 수 있는 루트 인증 기관"에 설치하면 됩니다. 필수는 아닙니다.

---

## 저장 동작 (설정 후)

- **HTTPS 접속**: 엑셀 저장 시 **매번** "다른 이름으로 저장" 창이 떠서
  원하는 폴더/파일명을 그때그때 고를 수 있습니다. (폴더를 기억하지 않음)
- **혹시 HTTP로 접속한 경우**: 예전처럼 브라우저 기본 다운로드 폴더로 저장됩니다.
  (서버 폴더에는 더 이상 저장되지 않습니다.)

---

## 다시 HTTP로 되돌리려면

`certs` 폴더(또는 그 안의 `server.key` / `server.crt`)를 지우고 서버를 재시작하면
자동으로 예전처럼 HTTP로 실행됩니다.

---

## 참고 (기술)

- 인증서 유무는 서버 시작 시 자동 감지합니다. (`server.js`)
  - `certs/server.key` + `certs/server.crt` 있으면 → HTTPS
  - 없으면 → HTTP (기존과 동일)
- `certs/` 폴더는 `.gitignore`에 등록되어 GitHub에 **업로드되지 않습니다.**
  개인키는 서버마다 로컬에서 생성하세요. (보안)
- WebSocket(실시간 갱신)은 HTTPS일 때 자동으로 `wss://`로 연결됩니다. (코드 수정 불필요)
